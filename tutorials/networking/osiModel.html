The OSI Model

by Jake Barnes

Posted on Wednesday 29, 2018 at 14:46 PM

This is a short post about what the OSI model is.

The Internet

So, just before we start with the OSI model, what is the 'Internet'? The Internet is essentially a network of networks - a huge set of computers/systems connected together. You will most likely have a small network in your home or office. A router, connected to a computer or mobile device through WiFi. That is a network. On your street you may have a local connector built by an internet service provider, this connects all the networks in all of the houses together. That box then goes off to another larger connector. Now just imagine all of the houses and all of the connections that exist out there in the world. That is the internet. It provides a way for 'Bob' in Tokyo to send digital messages to 'Alice' in London. The OSI model is a way of splitting up how the Internet works into smaller 'layers' in order to make sense of it all. The Internet is fantastic thing, but it is very very complex.

The OSI Model

As mentioned before, the internet has been 'split' into layers to control the complexity. One way of doing this (there are a few others) is by using the OSI Model. What this means is that the Internet is slit into 7 layers.


These are:

  • Physical
  • Data
  • Network
  • Transport
  • Session
  • Presentation
  • Application
With each layer doing something very important to enable a message to get form A to B.

Physical Layer

This layer is to do with physical medium that data passes through. Some examples would be ethernet, bluetooth, GSM, DSL. Equipment that can provide a physcial layer service include Network interface cards (NIC), Modems and Ethernet hubs.

Data Layer

This layer is to do with transfering data from one node to another within a network.

Network Layer

This layer is to do with the forwarding of 'packets' from a host to a destination. Some of the protocols that operate on the network layer include IPv4/v6, ICMP and IPSec.
